# Demo Screenshot Instructions

## Adding the Demo Screenshot

To complete the demo section in the README.md, you need to add a screenshot of the Streamlit interface.

### Instructions

1. **Run the Streamlit application:**
   ```bash
   streamlit run app.py
   ```

2. **Test the pipeline:**
   - Enter a programming problem
   - Let the complete pipeline run (classification → reasoning → coding → validation)
   - Wait for the final results to appear

3. **Take a screenshot:**
   - Capture the Streamlit interface showing the complete pipeline results
   - Include the problem input, generated solution, and validation results
   - Make sure the screenshot is clear and readable

4. **Save the screenshot:**
   - Save the image as `docs/demo.png`
   - Recommended size: 1200x800 pixels or similar
   - Format: PNG (preferred for clarity)

### Screenshot Content

The ideal screenshot should show:
- The problem input field with a sample programming problem
- The classification results
- The generated code solution
- The test results
- The validation status
- Any explanations or analysis provided

### File Location

Place the screenshot at:
```
docs/demo.png
```

This will automatically appear in the README.md demo section once the file is added to the repository.

### Tips

- Use a clear, concise programming problem for the demo
- Ensure all pipeline steps are visible in the screenshot
- Crop the image to focus on the relevant content
- Test the README.md locally to ensure the image displays correctly
