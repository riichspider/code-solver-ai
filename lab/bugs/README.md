# Bugs

**Purpose**: Track, document, and debug issues found during development and testing.

## Template for New Bug Reports

```markdown
# [Bug Title]

**Date**: YYYY-MM-DD  
**Status**: [open|investigating|fixed|wontfix]  
**Priority**: [low|medium|high|critical]

## Description
What happened vs what should happen.

## Steps to Reproduce
1. Step one
2. Step two
3. Step three

## Expected Behavior
What should have happened.

## Actual Behavior
What actually happened.

## Environment
- OS: 
- Python version:
- Related files:

## Root Cause
Why the bug occurred (if identified).

## Solution
How the bug was fixed.

## Prevention
How to avoid similar bugs in the future.
```

## Usage

1. Create a new file for each bug
2. Use descriptive filenames (e.g., `2024-05-06-cache-invalidation.md`)
3. Update status as you progress
4. Link related bugs together
